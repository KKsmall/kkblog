StringTokenizer类

  1. StringTokenizer类的定义

    StringTokenizer类是分隔字符串解析类，分隔符集(分隔令牌的字符)可以在创建时指定，也可以在每个令牌的基础上指定。
    StringTokenizer实例的行为有两种方式，取决于它是在创建时returnDelims标志的值为true还是false

  2. StringTokenizer几个属性

  3. 构造方法

    1. StringTokenizer(String str, String delim, boolean returnDelims)

      为字符串创建一个分隔解析器，分隔符是delim字符串，returnDelims如果为true,
      则除了返回被分隔的字符串的子串，分隔符也会作为穿插在子串之间的独立元素一起返回 关于上面unicode编码的相关补充介绍(摘用知乎的一个回答):最早的unicode字符空间只有65536个，utf16编码用两个字节足以表达所有字符，
      后来扩展到1765536之后，utf16就必须用4个字节才能表达所有字符。
      于是有些字符用2字节表示，有些就必须用4字节表示（如果全都用4字节表示，那就成了utf32了）。当处于同一个上下文中，既有4字节也有2字节时，你总得有个手段做区分吧。
      于是utf16编码就利用把题目中那块尚未分配字符的区域用来作为4字节扩展区域。你算一下就知道这个区域大小为2048，于是其中两个字节取值范围选取前1024既D800 - DBFF，
      另外两个取值范围选剩下的既DC00 - DFFF，这两个一组合，就能表达2^20大小的字符空间，既16      65536。
      所以2字节表示BMP，4字节表示剩下的平面，不多不少刚好17*65536，完美覆盖unicode字符空间

    2. StringTokenizer(String str, String delim)

      默认不返回分隔符

    3. StringTokenizer(String str)

      默认分隔符为空格、制表符(\t)、换行符(\n)、回车符(\r),也不返回分隔符

  4. 其他方法

    1. hasMoreTokens()

    2. nextToken()

      获取下一个token

    3. nextToken(String delim)

      返回指定分隔符的分隔串

    4. countTokens()

      计算有几个分隔串，如果构造方法有设值返回分隔符，则包括分隔符在内的。
